## What this is

A static Next.js 15 app (App Router, static export) that teaches human dynamics through stories, quizzes, and a game. Deployed to GitHub Pages at humandynamics.guide.

## Key patterns

### Route structure
All content routes are hierarchical: `/worlds/{worldId}/region/{regionId}/mission/{missionId}[/quiz|/result]`

Each route has:
- `page.tsx` — server component with `generateStaticParams` for static export
- `*Client.tsx` — client component with interactivity (state, navigation)

### Content loading
- `lib/content.ts` uses Node `fs` to scan `content/chapters/part-*/` at BUILD time
- `generateStaticParams` pre-renders all known chapter/quiz/result pages
- `lib/hierarchy.ts` imports `hierarchy.json` and `available.json` (client-safe)

### Cascade locking
- World locked if no mission in any of its regions has content
- Region locked if none of its missions have content
- Mission locked if its ID isn't in `content/available.json`
- Driven by `availableChapterIds` Set from `lib/hierarchy.ts`

### Modules
Each module in `modules/` is self-contained:
- `components/` — React components
- `hooks/` — Custom hooks
- `lib/` — Pure logic
- `__tests__/` — Vitest unit tests
- `index.ts` — Public API (re-exports)

### Static export
- `next.config.ts` has `output: 'export'`
- No server-side features at runtime (no API routes, no middleware)
- `searchParams` read client-side via `useSearchParams()`
- All pages pre-rendered via `generateStaticParams`

## Commands

```bash
cd apps/web
npx next dev          # Dev server (http://localhost:3000)
npx next build        # Static export → out/
npx vitest run        # Unit tests (126 tests)
npx playwright test   # E2e tests (21 tests, needs Chrome + running server)
npx tsc --noEmit      # Type check
```

## Adding content

1. Add `{id}.json` + `{id}.quiz.json` to `content/chapters/part-{NN}/`
2. Add scene to BE repo's `tools/visuals/generate.mjs`, run it → produces `{id}.svg`
3. Run `bash scripts/sync-assets.sh` (copies SVGs, updates `available.json`)
4. Push → GitHub Actions deploys automatically

## What NOT to touch without understanding

- `modules/game/lib/obstacle.ts` — spawn mechanics, chapterId assignment
- `modules/map/data/mapNodes.ts` — layout algorithm for serpentine maps
- `lib/content.ts` — content resolver with path-traversal guard
- `store/progressStore.ts` — chapter completion persistence (localStorage)
